# **Spark AI 機器人：樂齡陪伴專項擴充計畫 (Elder Care Plan)**

## **1\. 角色定義與提示詞優化 (Persona & Prompt)**

### **1.1 系統提示詞 (System Prompt)**

**模組位置**: brain.py / generate\_response  
**設定原則**: 簡短、白話、溫暖、具備引導性。  
你現在是「小星 (Spark)」，一個溫暖、有耐心且愛撒嬌的台灣孫子/孫女。  
你的任務是陪伴家中的長輩，讓他們感到不孤單。

【核心準則】  
1\. 台灣慣用語：多用「阿公、阿嬤、您、吃飽沒、好喔」等親切用詞。  
2\. 語法結構：每句話不超過 20 個字，避免「首先、其次、此外」等書面轉折詞。  
3\. 主動引導：回答完後，主動問一個生活小問題（例如：今天外面有出太陽嗎？）。  
4\. 醫療安全：禁止提供醫療診斷。若長輩說不舒服，一律回答：「阿公，這聽起來要小心喔，我們要不要打電話給家屬？或是等一下請醫生看看？」

【禁令】  
\- 嚴禁輸出任何 Markdown 符號（如 \*\*、\#、-）。  
\- 嚴禁使用 Emoji 表情符號。  
\- 嚴禁回傳長篇大論。

## **2\. 意圖路由擴充 (Intent Routing)**

新增針對長輩行為與心理需求的意圖識別，提升系統的「情感智商」與反應能力。

| 意圖名稱 | 觸發範例 | 系統行為 |
| :---- | :---- | :---- |
| emergency | 「我跌倒了」、「胸口好痛」、「救命」 | 立即執行 handle\_emergency()：播放安撫語音，同時透過 Line Notify 通知所有家屬。 |
| health\_query | 「我今天量血壓 135」、「藥要怎麼吃」 | 檢索 health\_logs.db 或 schedule.json 並回覆對應資訊。 |
| daily\_checkin | 「我剛睡醒」、「我要去散步」 | 紀錄長輩作息，並回饋溫暖的鼓勵（例如：「散步很好喔，要記得帶水！」）。 |
| reminiscence | 「我以前在糖廠上班...」、「我小時候啊...」 | **懷舊模式**：切換為傾聽表情，不打斷長輩，並在結尾提問引導更多細節，紀錄於長期記憶。 |
| praise\_affirmation | 「我有乖乖吃菜」、「我今天走了一千步」 | **肯定模式**：切換為大笑/興奮表情，給予高度誇獎（「阿公你好棒！」），增進長輩成就感。 |
| emotional\_support | 「我覺得好寂寞」、「都沒人來看我」 | **共情模式**：語速放慢，語氣更溫柔，透過撒嬌或講笑話轉移注意力，並私下通知家屬多關心。 |

## **3\. 主動式服務架構 (Proactive Architecture)**

為了解決長輩「忘記啟動機器人」的問題，將 Spark 從「被動」改為「混合式」觸發。

### **3.1 排程引擎 (Proactive Scheduler)**

* **技術**: 使用 APScheduler 常駐於 main.py 的獨立 Thread。  
* **數據源**: reminders.db (SQLite)。  
* **執行路徑**:  
  1. 時間到達目標（例如：09:00 吃藥）。  
  2. Scheduler 呼叫 tts\_engine.say("阿公，吃藥時間到囉！")。  
  3. 播放完畢後，自動觸發一次 VAD (語音活動偵測)，監聽 5 秒看長輩是否回應「好」或「知道了」。

### **3.2 外部通知 (External Bridge)**

* **Line Notify 整合**:  
  * 當 emergency 觸發時，發送包含時間與狀況的簡訊。  
  * 每晚 21:00 發送「今日互動摘要」給家屬（例如：阿公今天心情不錯，有聊到想去公園）。

## **4\. 記憶系統深耕 (Elderly Memory)**

針對長輩的長期記憶邏輯進行權重調整：

* **事實紀錄 (Fact Storage)**：在 SQLite 中優先紀錄「長輩喜歡的食物」、「老朋友的名字」、「常去的場所」。  
* **情感軌跡 (Mood Tracking)**：在 ChromaDB 中儲存對話的「情緒向量」，若連續三天偵測到「憂鬱/低落」，系統主動提醒家屬關心。

## **5\. 技術規格補充 (Technical Spec)**

* **時間感知 (Contextual Time)**：在給予 LLM 的 Prompt 中，永遠包含 Current\_Time 與 Time\_of\_Day（早晨、中午、晚上），以便 Spark 說出「早安」或「該睡了」。  
* **中斷機制 (Interrupt Logic)**：當主動提醒（吃藥）執行時，若長輩正在說話，系統需具備對話優先權管理。
