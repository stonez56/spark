# 🐈 Mimo 溫感貓咪助理 — 產品需求文件 (Product Requirements Document, PRD)

---

## 1. 產品概述 (Product Overview)
**Mimo** 是一款專為桌面與家庭場景設計的「AI 溫感療癒貓咪助理」。它擺脫了傳統老人照護機器人的沉悶感，改以「傲嬌、聰明、極具療癒感且愛碎碎念」的台灣腔貓咪人設，結合實體樹莓派硬體，與使用者進行高互動性的對話、觸控、物理轉頭追蹤及視覺分析。

*   **專案時程**：必須於 **6 月底前** 完成全部軟硬體開發與整合。
*   **核心目標**：高可行性、低複雜度、高度趣味性與極佳的視覺美學。

---

## 2. 硬體系統配置 (Hardware Architecture)
本專案基於 **樹莓派 5 (8GB)** 進行開發，以確保本地 LLM 與電腦視覺模組以極低延遲流暢運行。

### 實體硬體接線清單 (Pin Configuration)：
1.  **大腦**：樹莓派 5 (8GB)，使用 USB-C 5V 5A 供電。
2.  **螢幕 (OLED SSD1306, 0.96")**：連接至樹莓派 I2C 介面。
    *   `VCC` -> 3.3V (Pin 1)
    *   `GND` -> GND (Pin 9)
    *   `SDA` -> SDA (GPIO 2, Pin 3)
    *   `SCL` -> SCL (GPIO 3, Pin 5)
3.  **觸碰感測器 (Touch Sensor)**：貼於貓咪頭頂。
    *   `VCC` -> 3.3V
    *   `GND` -> GND
    *   `SIG` -> GPIO 17 (Pin 11)
4.  **雙軸舵機雲台 (SG90 x2)**：作為頸部轉動模組。
    *   **Pan (左右旋轉)** -> GPIO 18 (Pin 12)
    *   **Tilt (上下/歪頭)** -> GPIO 19 (Pin 35)
5.  **眼睛 (Pi Camera IMX219)**：固定於雲台鏡頭架上，透過 CSI 軟排線連接至樹莓派 5 的 CAM/DISP 連接埠。
6.  **嘴巴 (USB 喇叭)**：直插樹莓派 USB 連接埠，隨插即用。

## 3. 軟體功能需求 (Functional Requirements)

### 3.1 傲嬌貓咪認知與意圖系統 (Brain & Cognitive)
*   **Mimo 貓咪個性**：
    *   對話使用親切自然、充滿台灣口語習慣的繁體中文。
    *   稱呼使用者為「主人」，自稱為「本喵」或「Mimo」。
    *   性格高傲、傲嬌但極度關心主人。回答簡短（每句限 20 字內），句尾隨機帶有「喵～」、「哼」。
    *   具備基本醫療守護底線，若主人表示不舒服，會緊張炸毛並提醒就醫。
*   **意圖路由器擴充**：
    *   `chat`：一般貓咪碎碎念對話。
    *   `pet_cat`：使用者點擊 UI 或觸碰實體貓頭時觸發。
    *   `temp_analysis`：當量測或模擬體溫時進行傲嬌吐槽。
    *   `take_photo`：控制雲台轉正，啟動 Pi Camera 拍照並透過 Vision 大腦分析。

### 3.2 貓咪專屬語音快取 (Audio Cache)
*   預先合成 45 個貓咪專屬台詞（Piper TTS）。
*   在 LLM 運算（Thinking 狀態）時，即時播放對應的 Filler 貓咪音效，消除等待焦慮。

### 3.3 主動人臉偵測、主動喚醒與雲台追隨 (Active Vision & Gimbal Tracking)
*   **背景 OpenCV 偵測線程**：
    *   在 `Idle` 狀態下，背景線程每 1.5 秒透過 Pi Camera 擷取超低解析度畫面進行 Haar Cascade 人臉偵測。
    *   **主動喚醒**：當偵測到人臉出現，Mimo 表情點亮，頭部抬起，主動說出歡迎主人的台詞。
    *   **即時物理追蹤**：當偵測到人臉移動，計算畫面中心偏離值，透過 PWM 控制 GPIO 18/19 舵機，讓貓頭與鏡頭實時跟隨人臉擺動！
    *   **自動休眠**：若連續 2 分鐘未偵測到人臉，頭部緩緩垂下進入打盹省電狀態。

### 3.4 視覺美學 UI (FastAPI & Web UI)
*   **色彩美學**：極致現代的深靛藍（Midnight Indigo）與極光粉（Aurora Pink）、霓虹綠的玻璃擬態風格。
*   **貓咪 SVG 臉部**：
    *   在 SVG 臉部追加**貓耳**與**貓鬚**。
    *   為貓耳編寫 CSS 微動畫（如聆聽時耳朵抖動、思考時轉為向後的「飛機耳」）。
*   **虛擬溫度與擼貓面板**：
    *   包含一個「擼貓」按鈕，觸發後發送 WebSocket 訊號。
    *   包含一個體溫模擬滑動條，模擬體溫量測並將數據送給大腦進行分析。

### 3.5 大腦智慧優化與記憶隔離防污染機制 (Brain Intelligence & Memory Isolation)
*   **優質中文模型鎖定 (Strict Chinese LLM Pinning)**：
    *   雲端大腦首選鎖定在 OpenRouter 表現最為頂尖且快速的免費繁體中文模型 **`qwen/qwen3-next-80b-a3b-instruct:free`** (Qwen 3 Next 80B)。其具備卓越的繁體中文語境、台灣腔調適配與極高的指令理解能力。
*   **多級備用大腦重試與本地退路機制 (Multi-Level Brain Retry & Local Fallback)**：
    *   **雲端備用重試鏈**：系統引進精簡重試機制。首選為 Qwen 3 80B，次選為 Llama 3.3 70B。若首選遭遇 404 等錯誤，系統會自動切換至備用雲端大腦。
    *   **極速限流跳轉 (Snappy 429 Bypass)**：若偵測到 OpenRouter 免費端點返回 **`429` (Rate Limit 上游限流)**，為了不讓使用者在前端白白等待 10 多秒的重複超時，系統會**立刻中斷雲端重試，在 0.1 秒內直接降級至本地大腦運作**，保障使用者體驗極致流暢！
    *   **本機終極退路**：當雲端完全失效或被限流時，本地運行的 Ollama (`gemma3:1b` / `llama3.2:3b`) 將無縫接管，在 1 秒內輸出高度合理且具備貓咪人設的繁體中文對話。
*   **生成穩定度溫度限制 (Temperature Constraint)**：
    *   在 OpenRouter (`_cloud_chat`) 與本地 Ollama (`_local_generate`) 請求中，強制限制解碼參數 `temperature=0.3`。低溫解碼能迫使 LLM 產生高連貫性、高穩定性的文字流，杜絕大語言模型隨機胡言亂語與出現不符人設的英語混雜。
*   **記憶資料庫物理隔離 (Memory Database Physical Isolation)**：
    *   為防止 Spark 舊有的「阿公與小星」對話紀錄被語義檢索提取，進而嚴重污染 Mimo 的貓咪人設大腦，將 SQLite 與向量資料庫重命名為獨立的 **`mimo_memory.db`** 與 **`mimo_chroma_db`**（向量集合名稱更名為 `mimo_conversations`）。
    *   在角色變更時，自動移除所有舊專案遺留之資料庫暫存，實行乾淨的「記憶物理牆隔離」。

## 4. 系統狀態機 (State Machine)
Mimo 的實體頭部與網頁表情將與以下狀態深度整合：
*   **Idle**：貓咪打盹/眨眼。若人臉偵測啟動，會主動跟隨。
*   **Listening**：耳朵如天線般前後微幅抖動，呈現專注大眼睛。
*   **Thinking**：耳朵向後貼呈「飛機耳」，頭部左右微幅「歪頭殺」，眼睛呈現旋轉或疑惑波浪。
*   **Speaking**：嘴巴隨聲音震幅開合，耳朵與鬍鬚隨之輕微抖動。
*   **Petting**：眼睛變為愛心 (`^ _ ^`)，頭部微微上揚，播放呼嚕聲。
*   **Angry**：體溫異常或主人太久沒理它時，眼睛豎直。
